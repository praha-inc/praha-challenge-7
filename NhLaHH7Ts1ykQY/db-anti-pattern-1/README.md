## 課題1
一つのカラム内にカンマ区切りで格納する設計は以下のような問題があります。

- パターンマッチによる比較になるため、等価性が担保できない
    - 例えば、tag=5に合致するデータを引っ張って来たいとき、「"5,"」、「",5,"」、「",5"」の3つをLIKEに入れる必要がある
- タグに紐づくテーブルと結合したい際に、クエリが煩雑になる
- 特定の値をもつ行の更新が煩雑
- データを追加する際に、妥当性の検証が難しい
    - 区切り文字の種類やタグにふさわしくないものまで混在する可能性がある
